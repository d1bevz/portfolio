# Лог решений

> Каждое решение — купил / не купил / продал / отложил — с датой и обоснованием.
> Это память портфеля и защита от импульсивных решений на новостях.

Формат записи:

```
## YYYY-MM-DD — <действие> <тикер>
- **Решение:** купил / пропустил / продал / в watchlist
- **Сумма:** € (если сделка)
- **Тезис:** почему
- **Данные на момент решения:** цена X € (источник, дата), ключевые цифры
- **Уровень уверенности:** high / medium / low
- **Что может пойти не так:** риски
```
